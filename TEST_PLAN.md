# خطة اختبار نظام المكتبة

## نطاق الاختبار
- واجهة المستخدم (frontend)
- واجهات الـ API (backend)

## حالات اختبار أساسية
1) عرض قائمة الكتب بنجاح.
2) إضافة كتاب جديد والتأكد من ظهوره في القائمة.
3) تعديل بيانات كتاب والتأكد من التحديث.
4) حذف كتاب والتأكد من اختفائه.
5) استعارة كتاب وتغيير حالته إلى مستعار.
6) إرجاع كتاب وتغيير حالته إلى متاح.
7) البحث عن كتاب بالعنوان أو المؤلف.

## بيانات اختبار مقترحة
- كتاب: "Introduction to Algorithms" — المؤلف: "Cormen"
- كتاب: "Clean Code" — المؤلف: "Robert C. Martin"
