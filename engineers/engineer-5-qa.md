# تعليمات المهندس 5 (الاختبار وضمان الجودة)

## بيانات المهندس
- الاسم: مصطفى ادلبي
- الدور: الاختبار وضمان الجودة
- تاريخ البدء: [اكتب التاريخ]

## الهدف
اختبار النظام يدويًا والتحقق من واقعية النتائج.

## الفرع المقترح
`qa/testing`

## المهام المطلوبة
1) تحديث `TEST_PLAN.md` ليتضمن:
   - حالات الاستعارة/الإرجاع
   - تواريخ الاستعارة والانتهاء
   - حساب المتبقي والتأخير
2) تنفيذ اختبار يدوي على الواجهة.
3) توثيق الملاحظات داخل `DOCS.md` أو عبر Jira.
4) تحديث مهمة Jira الخاصة بالاختبار وتحريكها حسب الحالة.

## الكود الذي يجب رفعه
- `TEST_PLAN.md`
- `DOCS.md` (إن لزم)

## خطوات الرفع كاملة
```text
git clone https://github.com/alasermohamad94/library_management.git
cd library_management
git checkout -b qa/testing
git add TEST_PLAN.md DOCS.md
git commit -m "Update test plan and QA notes"
git push -u origin qa/testing
```
ثم افتح Pull Request على `main`.

## خطوات العمل التفصيلية
1) استلم رابط المستودع من قائد الفريق.
   - إذا كان لديك نسخة قديمة، احذفها وأعد clone من المستودع الجديد.
2) نفّذ clone وانتقل للمجلد.
3) أنشئ فرع الاختبار.
4) حدّث `TEST_PLAN.md` وأضف الملاحظات في `DOCS.md`.
5) Commit واضح ثم Push.
6) افتح Pull Request مع وصف قصير.
7) أضف رابط الـ PR داخل مهمة Jira الخاصة بك.

## مخرجات إثبات العمل
- رابط Pull Request.
- خطة اختبار واضحة ومحدّثة.
